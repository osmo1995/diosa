## üêõ Known Issues & Workarounds

### **Issue 1: BeforeAfterSlider Causes Loading Hang**
- **Symptom:** Site stuck on "LOADING..." forever
- **Cause:** BeforeAfterSlider component in hero section
- **Solution:** Use simple `<OptimizedImage>` instead
- **Status:** FIXED (reverted to simple hero)

### **Issue 2: Gemini Quota Exhaustion**
- **Symptom:** `429 RESOURCE_EXHAUSTED` when generating images
- **Cause:** Daily quota limit hit
- **Solution:** 
  - Wait 24 hours for reset, OR
  - Enable billing in Google AI Studio, OR
  - Switch to `models/gemini-2.5-flash-image` (often has separate quota)
- **Status:** Workaround implemented (model switching)

### **Issue 3: HuggingFace API Deprecated**
- **Symptom:** `410 Gone` from api-inference.huggingface.co
- **Cause:** Free inference API shut down
- **Solution:** Code supports `--provider=hf` but won't work without paid endpoint
- **Status:** Documented, not blocking

### **Issue 4: Multiple Reverts Caused Layout Confusion**
- **Symptom:** Wrong sizing, layout broken after reverts
- **Cause:** Reverted to wrong commit (too old)
- **Solution:** Correct version is commit `7455fd7` (before slider, after layout improvements)
- **Status:** FIXED (restored correct version)

## üìû Getting Help

### **If Site Won't Load:**
1. Check Vercel deployment logs: https://vercel.com/anas-projects-37a977a8/diosa-yorkville
2. Check browser console for errors (F12)
3. Verify `dist/` folder was generated: `ls dist/`
4. Try local build: `npm run build && npm run preview`

### **If Images Missing:**
1. Check if assets exist: `ls public/generated/*/`
2. Verify `data/generatedImages.ts` maps them correctly
3. Regenerate if needed: `node scripts/generateAllAssets.mjs`

### **If Style Generator Broken:**
1. Check `/api/style` endpoint responds
2. Verify `GEMINI_API_KEY` in Vercel environment variables
3. Check Gemini API quota hasn't exhausted
4. Test with `--dry-run` flag first

### **If Build Fails:**
1. Run `npx tsc --noEmit` to find TypeScript errors
2. Check `npm install` completed successfully
3. Verify Node.js version (need v20+)
4. Clear cache: `rm -rf node_modules dist && npm install`
